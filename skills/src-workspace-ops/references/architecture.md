# System Architecture

## Overview

A D3I data donation study runs on a single SURF Research Cloud (SRC) workspace. The workspace hosts two main applications:

1. **mono** — An Elixir/Phoenix web application (the "host") that manages the study, authentication, and data storage
2. **data-donation-task** — A React/TypeScript workflow application (the "workflow") that runs inside an iframe in mono and handles the actual data donation interaction with participants

## Iframe Communication Protocol

The workflow (data-donation-task) runs inside an iframe. Communication uses the browser MessageChannel API:

### Setup sequence

1. mono renders a LiveView page with `<iframe>` and `phx-hook="FeldsparApp"`
2. `feldspar_app.js` hook creates a `MessageChannel` (port1 + port2)
3. Hook sends `"live-init"` action to iframe via `postMessage`, transferring port2
4. Workflow's `live_bridge.ts` receives port2 and stores it for sending commands
5. Hook also sends the current locale to the workflow

### Data flow

```
Participant interacts with workflow UI
  → Workflow builds CommandSystemDonate {key, json_string}
  → live_bridge.ts sends via port.postMessage()
  → feldspar_app.js receives on port1
  → Pushes "feldspar_event" to LiveView
  → LiveView handler routes to storage backend
  → Data persisted (Research Drive / S3 / LocalFS)
```

### Command types

| Command | Fields | Purpose |
|---------|--------|---------|
| `CommandSystemDonate` | `key`, `json_string` | Submit donation data |
| `CommandSystemExit` | `code`, `info` | Signal workflow completion |

## Build and Deployment

The `self-d3i:latest` Docker image is **built from source during Ansible provisioning**, not pulled from a registry. The Ansible playbook in `d3i-infra/researchcloud-items` (`playbooks/deploy-next.yaml`) clones `d3i-infra/mono`, runs `docker build` in `mono/core/`, and produces the image. After the build completes, the source is no longer present on the workspace — only the built image remains.

The `docker-compose.yaml` at the filesystem root is also **generated by Ansible** from the template in `d3i-infra/researchcloud-items` (`playbooks/deploy-next.yaml`). It is not part of the mono repo.
